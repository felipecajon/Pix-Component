Eventos

generateCode:

Irá criar o QrCode

Parametros

1º (String): Texto / Código que dará origem ao QrCode.

2º (Object): Configurações opcionais

  • size: (default | 300) Tamanho em px que o QrCode deverá ser exibido
  • setInterval: (default | 2000) Tempo ( em milisegundos ) entre as futuras requisições sobre o status do pagamento
  • url: (default | http://localhost:3000/response.json) API de consulta de status.

OBS: São 3 os possiveis status que devem retornar da API: true, false ou waiting. Para fins didaticos há um arquivo chamado response.json neste exemplo que simula a resposta do servidor, deixe-o sempre com o status 'waiting' para ver que o front fará varias requisições, enquanto isso altere o status para um dos outros 2 possiveis status simbolizando uma troca de resposta pelo servidor.
